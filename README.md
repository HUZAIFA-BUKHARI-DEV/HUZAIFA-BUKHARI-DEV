## Engineering Principles

```text
01. Build for maintainability before cleverness.

02. Keep API behavior predictable.

03. Put business logic in clear, testable boundaries.

04. Design databases around real workflows, not temporary UI screens.

05. Optimize queries before scaling infrastructure.

06. Prefer simple architecture until complexity becomes necessary.

07. Treat validation and authorization as first-class concerns.

08. Automate repetitive operational work.

09. Make failures observable and debuggable.

10. Write code that another developer can safely continue.
```

## How I Approach Backend Work

### 1. Understand the workflow

Before writing code, I identify:

- The users involved
- Their permissions
- The business states
- The data relationships
- The failure conditions
- The expected API behavior

### 2. Design the database

I focus on:

- Clear table responsibilities
- Correct foreign keys
- Useful indexes
- Controlled nullable fields
- Consistent timestamps
- Reliable migration history

### 3. Define the API contract

I define:

- Request structure
- Validation rules
- Authorization requirements
- Success responses
- Failure responses
- Pagination and filters

### 4. Implement business logic

I prefer keeping large workflows out of controllers through:

- Actions
- Services
- Jobs
- Policies
- Events
- Listeners
- Resource classes

### 5. Prepare for production

I verify:

- Environment configuration
- Logging
- Queues
- Cron jobs
- File permissions
- Storage links
- Deployment paths
- Timezone behavior
- Failure recovery
